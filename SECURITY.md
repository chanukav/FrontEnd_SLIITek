# Security Policy

## Supported Versions

We actively monitor and provide security updates for the following versions of the software:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |
| < 1.0   | No        |

## Reporting a Vulnerability

We take the security of this project seriously. If you find a security vulnerability, please do **NOT** open a public issue. Instead, report it privately so we can address it securely.

To report a vulnerability, please email the maintainers or project administrators directly. 

When reporting, please include:
- A description of the vulnerability and its potential impact.
- Detailed step-by-step instructions (and code/payload if applicable) to reproduce the issue.
- Details of the environment where it was tested.

We will acknowledge receipt of your report within 48 hours and work with you to patch the issue before making it public.
