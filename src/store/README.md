# State Management (Store)

This directory is reserved for global state management configuration and logic.

## Purpose

- Store the application's global state (e.g., Redux store, Zustand store, or React Context).
- Manage actions, reducers, and slices for state updates.
- Centralize data that needs to be accessed across multiple components.

## Examples

- `useAuthStore.js`: Zustand store for authentication state.
- `store.js`: Redux store configuration.
- `slices/`: Directory for Redux slices.
